Support for Response Presets

Cr4O13

GCI Home v1.0 BETA 7 introduces ‘Response Presets’ to simplify the configuration of an axis with output response specifications.

Response presets support reuse of responses and help keeping the axis configuration readable.

Before BETA 7 an axis output response had to be specified literally in the axis spec:

{ "axis": [
  { "id": [ 2, "Pedal Rotation" ], 
    "write": [ "RUDDER POSITION", "Position", { 
        "response": [
          [ -1.000, -1.000 ], 
          [ -0.924, -0.836 ], 
          [ -0.838, -0.687 ], 
          [ -0.740, -0.552 ], 
          [ -0.629, -0.429 ], 
          [ -0.501, -0.314 ], 
          [ -0.355, -0.206 ], 
          [ -0.189, -0.102 ], 
          [  0.000,  0.000 ], 
          [  0.189,  0.102 ], 
          [  0.355,  0.206 ], 
          [  0.501,  0.314 ], 
          [  0.629,  0.429 ], 
          [  0.740,  0.552 ], 
          [  0.838,  0.687 ], 
          [  0.924,  0.836 ], 
          [  1.000,  1.000 ]
        ]
      }
    ] 
  }
}

Response Preset Definition

Starting from BETA 7 the response preset itself must be defined in a defaults ‘responses’ specification with a preset name and the response settings:

"defaults": {
  "responses": {
    "flight-controls": [
      [ -1.000, -1.000 ], 
      [ -0.924, -0.836 ], 
      [ -0.838, -0.687 ], 
      [ -0.740, -0.552 ], 
      [ -0.629, -0.429 ], 
      [ -0.501, -0.314 ], 
      [ -0.355, -0.206 ], 
      [ -0.189, -0.102 ], 
      [  0.000,  0.000 ], 
      [  0.189,  0.102 ], 
      [  0.355,  0.206 ], 
      [  0.501,  0.314 ], 
      [  0.629,  0.429 ], 
      [  0.740,  0.552 ], 
      [  0.838,  0.687 ], 
      [  0.924,  0.836 ], 
      [  1.000,  1.000 ]
    ]
  }
}

Once defined, the preset name can be used in multiple axis specifications as the value of the response field:

{ "axis": [
    { "id": [ 2, "Pedal Rotation" ], 
      "write": [ "RUDDER POSITION", "Position", { "response": "flight-controls" } ]
    }
  ]
}

Response Presets File

To assist in the management of response presets, additionally to the above, the response presets can be moved to a separate JSON configuration file (e.g. “responses.json”). The file must reside in the resources folder.

{ "flight-controls": [
    [ -1.000, -1.000 ], 
    [ -0.924, -0.836 ], 
    [ -0.838, -0.687 ], 
    [ -0.740, -0.552 ], 
    [ -0.629, -0.429 ], 
    [ -0.501, -0.314 ], 
    [ -0.355, -0.206 ], 
    [ -0.189, -0.102 ], 
    [  0.000,  0.000 ], 
    [  0.189,  0.102 ], 
    [  0.355,  0.206 ], 
    [  0.501,  0.314 ], 
    [  0.629,  0.429 ], 
    [  0.740,  0.552 ], 
    [  0.838,  0.687 ], 
    [  0.924,  0.836 ], 
    [  1.000,  1.000 ]
  ],
    ...
}

In the defaults responses specification can reference the file name:

"defaults": {
  "responses": "responses.json"
}

All response presets defined in the referred file will be available for use in the axis specifications.
